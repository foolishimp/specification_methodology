# Release v1.0.0

**Repository**: `specification_methodology`
**Tag**: `v1.0.0`
**Release Commit**: `43ce5c8`
**Release Date**: 2026-04-15

## Position

`v1.0.0` is the first explicit released cut of the shared specification
methodology as its own source-of-truth repository.

This release establishes the methodology library as an independent product
rather than a borrowed surface inside another product repository.

## Included In This Release

- `SPEC_METHOD.md` tightened as the baseline constitutional method
- recursive product taxonomy raised into the baseline method
- source-versus-installed methodology authority made explicit
- `WORLD_MODEL_METHOD.md` added as the constitutional refinement for
  world-model construction work
- `GLOSSARY_GUIDE.md` added as the shared terminology surface across recursive
  product, graph-native, and world-model work
- `GRAPH_METHOD.md`, `RELEASE_METHOD.md`, and `TICKET_METHOD.md` repriced to
  align with the baseline taxonomy and glossary
- starter templates repriced to conform to the active requirement and tenant
  classification rules

## Key Outcomes

This release makes the methodology corpus materially more coherent in five
places:

1. recursive product terms are now explicit and load-bearing
2. world-model work has its own named constitutional method and companion guide
3. ticket tracking now records the triage metadata required by the baseline
   method
4. installed-versus-source methodology authority is singular and explicit
5. templates no longer seed method-invalid requirement and tenant surfaces

## Release Boundary

The tag `v1.0.0` identifies the tapped methodology cut at commit `43ce5c8`.

This note was added immediately after the tag was tapped so the repository has
a durable human-readable release surface. The release note is therefore a
post-tag documentation addition on `main`, not part of the tagged tree itself.
