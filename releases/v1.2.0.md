## Material Changes

`SPEC_METHOD.md` now makes the probabilistic work boundary explicit. The method
governs declared work contracts, admissible evidence, provenance, and control
truth. It does not absorb worker-internal reasoning or domain HOW into the
framework. It also publishes the testing strategy taxonomy that separates
design/module conformance tests from UAT/acceptance tests, with harnessed and
live sandbox lanes.

`GLOSSARY_GUIDE.md` now defines the testing authority terms and graph-native
compute terms needed by the tightened standards: design/module conformance
test, unit test, UAT/acceptance test, sandbox test, harnessed sandbox test,
live sandbox test, edge traversal, probabilistic compute, `F_D`, and `F_P`.

`ODD_METHOD.md` now states that an ODD product must use its own edge-traversal
unit of compute to build operative framework behavior. The required order is:

```text
outcome traversal first
declarative GTL/ABG carrier structure second
minimal imperative adapter code last
```

The method now treats imperative service ownership of operative transitions,
workspace orchestration described after the fact with GTL vocabulary, and
imperative closure/continuation/lineage decisions outside declared carrier
truth as method violations.

`DESIGN_MODULE_METHOD.md` now aligns explicitly with `ODD_METHOD.md`. It
governs deterministic realization structure inside or beneath ODD carriers. It
does not authorize replacing graph functions, edge traversals, or GTL modules
with deterministic implementation modules.

`TICKET_METHOD.md` now requires multi-build-tenant independence for active and
backlog execution work. It also adds library/commonization declaration for
implementation migration tickets so recurring realization patterns are not
rebuilt locally by drift.

`WORLD_MODEL_METHOD.md` now sharpens Markov-object publication language. The
published Markov-object cut is the primary semantic unit for world-model
construction, publication, and composition. The method also clarifies the
candidate epistemic status of the empirical Markov-object program and aligns
world-model boundary language with `DESIGN_MODULE_METHOD.md`.
