## Position

`v1.3.0` is the tapped UX-method constitutional authority cut.

This release promotes `UX_METHOD.md` from draft to active and tightens it so
adopted UX surfaces cannot treat existing code, framework convention, local
ADRs, scaffold behavior, comments, screenshots, or generated artifacts as rival
realization authority.

## Included In This Release

This release includes updates to:

- `specification/standards/UX_METHOD.md`

## Material Changes

`UX_METHOD.md` now states that, when adopted by a product, ticket, design
module, or review, it is the sole UX realization authority for the covered UX
surface.

The method now makes the UX/ODD boundary explicit:

- UX surfaces are projection and control surfaces.
- Product-truth-changing UX messages must map to admitted product carriers.
- The UX layer must not become the hidden constructive carrier.
- If no carrier exists for a product-truth-changing message, the surface is
  blocked until the carrier is authored or the requirement is repriced.

The method now requires each UX surface to declare:

- `State`
- `Msg`
- `Update`
- `Cmd`
- `Sub` where subscriptions exist

Each `Cmd` declaration must name its effect kind, typed input, target boundary,
success message, failure message, and deterministic correlation key where
needed for replay.

The method now turns Msg replay into an executable proof obligation. Product UX
surfaces must carry replay proofs over initial state, ordered messages,
expected final state, expected commands, command-return messages, and injected
deterministic values for time, randomness, generated ids, and external payloads.

The method now tightens `AssetSurface` binding:

- product-truth-changing messages map one-to-one to action-registry entries or
  a documented no-semantic-change wrapper
- absence of an `AssetSurface` requires an equivalent admitted typed contract
- absence of any admitted contract blocks product-truth-changing UX actions

Runtime validation is now mandatory at untrusted or out-of-process seams,
including HTTP/RPC, MCP, file-loaded records, websocket/event-stream payloads,
agent/worker outputs, and URL/storage/clipboard/browser integration payloads.
Compile-time TypeScript alone is not sufficient at those seams.

The method now requires UX adoption to include the
`DESIGN_MODULE_METHOD.md` Irreducible Architectural Carrier Set and structural
carrier diagram required for design-method closure.

The method now adds scaffold and exemption law. A UX surface is governed by
default when it is reachable from the product shell, used as proof, mutates
product truth, dispatches work, or changes Context, selection authority,
closure, continuation, evidence, or provenance. Scaffolds and debug surfaces
must be explicitly exempted, bounded, labeled where user-facing, assigned a
retirement condition, and prevented from release closure if they remain.
